# Use Case Diagram

## Project: Smart Cab Security Platform

### Objective
The Use Case Diagram illustrates how different users interact with the Smart Cab Security Platform. It shows the main functionalities available to passengers, drivers, and administrators.

### Actors
- Passenger
- Driver
- Admin

### Use Cases

#### Passenger
1. Register/Login
2. Book Cab
3. Track Ride
4. Verify Driver Details
5. Send Emergency Alert
6. View Ride History

#### Driver
1. Register/Login
2. Accept Ride Request
3. Update Ride Status
4. Manage Profile

#### Admin
1. Manage Users
2. Monitor Security Alerts
3. Manage Drivers
4. Generate Reports

### Description
The Passenger can register, log in, book cabs, track rides, verify driver information, and send emergency alerts during emergencies. The Driver can accept ride requests, update ride status, and manage profile information. The Admin manages users and drivers, monitors security alerts, and generates system reports. These interactions together ensure a secure and efficient cab service platform.
